A · Kacheln Plus

Weiterentwicklung des heutigen Dashboards: gleiches Raster (¼/½/voll), aber alle Module als Widgets verfügbar – neu: Aufgaben mit Mail-Vorschlägen, Finanzen kombiniert, Datacenter, Berichte, Admin-Kacheln. Geringster Umbauaufwand, vertraute Bedienung.

B · Cockpit

Tagesorientiert: KPI-Leiste oben, laufender Timer prominent, links „Mein Tag“ (Aufgaben + Termine + Mail-Vorschläge), rechts schmale Info-Spalte. Beantwortet „Was ist heute wichtig?“ auf einen Blick. Mittlerer Aufwand.

C · Zonen

Thematische, einklappbare Zonen („Mein Tag“, „Finanzen & Projekte“, „Daten & Verwaltung“) plus Favoriten-Leiste mit Schnellaktionen. Skaliert am besten, wenn künftig weitere Module dazukommen. Höchster Aufwand.

📐 Endgeräte

  • iPhone (<640px): 1–2 Spalten, Favoriten/KPIs horizontal scrollbar, Timer als Sticky-Leiste.
  • iPad (768–1024px): 2 Spalten, Cockpit-Seitenspalte rutscht nach unten.
  • MacBook/Desktop (>1024px): volles Raster, max. Breite ~1400px zentriert.

👤 Individuell je Benutzer

  • Konfiguration wandert von localStorage → Backend (JSON-Feld je Benutzer, z. B. GET/PUT /api/users/me/dashboard) → gleiches Dashboard auf allen Geräten.
  • Rollenbasierter Widget-Katalog: Admin-Widgets (Buchhaltung, Benutzer, System) nur für Admins.
  • Bestehende localStorage-Config wird beim ersten Login automatisch übernommen (Migration).
  • „Anpassen“-Modus: Widgets wählen, Größe (¼/½/voll), Reihenfolge per Drag&Drop – wie bisher, nur serverseitig gespeichert.
